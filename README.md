# LUXE MAISON — интернет-магазин одежды

Портфолио-проект: витрина одежды и обуви с корзиной, оформлением заказа,
аккаунтами, избранным, фильтрами/поиском, 3 языками интерфейса (RU/EN/AZ)
и светлой/тёмной темой. Бэкенд — Node.js/Express, база данных — SQLite
(один файл, без отдельного сервера БД).

## Как запустить

1. Установи [Node.js](https://nodejs.org) (LTS-версия), если ещё не установлен.
2. Открой терминал в папке проекта.
3. Установи зависимости:
   ```
   npm install
   ```
4. Заполни базу тестовыми товарами:
   ```
   npm run seed
   ```
5. Запусти сервер:
   ```
   npm start
   ```
6. Открой в браузере: `http://localhost:3000`

Если порт занят или нужен другой — поменяй значение `PORT` в файле `.env`.

## Структура проекта

```
backend/
  server.js          — точка входа, здесь всё "склеивается"
  db/
    database.js       — подключение к SQLite
    schema.sql         — описание таблиц
    seed.js             — тестовые товары (36 штук, 9 категорий)
  routes/
    products.js         — каталог, фильтры, поиск
    cart.js               — корзина
    orders.js              — оформление заказа (тестовое, без оплаты)
    auth.js                  — регистрация/вход/выход
    wishlist.js                — избранное
  middleware/
    auth.js                      — определение гостя/пользователя, защита роутов

frontend/
  index.html, catalog.html, cart.html,
  login.html, register.html, account.html, wishlist.html
  css/style.css        — все стили сайта (светлая/тёмная тема через CSS-переменные)
  js/                  — по одному файлу на страницу + общие (theme, i18n, common)
  locales/             — словари переводов ru.json, en.json, az.json
```

## Как всё устроено (коротко)

- **База данных** — это файл `database.db`, который появится в корне проекта
  после первого запуска. Используется встроенный в сам Node.js модуль
  `node:sqlite` (начиная с версии 22.5) — не нужно ничего компилировать,
  никаких Visual Studio / build tools, просто `npm install` и всё работает.
  Можно открыть `database.db` программой типа DB Browser for SQLite,
  чтобы посмотреть таблицы и данные глазами. Node выведет предупреждение
  `ExperimentalWarning: SQLite is an experimental feature` при запуске —
  это нормально, не ошибка, просто модуль относительно новый.
- **Корзина** работает и без регистрации — гостю присваивается временный id
  через cookie (сессию). Как только гость регистрируется или входит —
  его гостевая корзина "переезжает" на аккаунт.
- **Пароли** хранятся не как текст, а в хэшированном виде (bcrypt) —
  даже если кто-то откроет файл базы, паролей он не увидит.
- **Checkout тестовый** — заказ реально сохраняется в базу (можно посмотреть
  в личном кабинете), но без подключения настоящей оплаты.
- **Переводы** — в `frontend/locales/*.json` лежат словари, `js/i18n.js`
  подставляет нужный текст во все элементы с атрибутом `data-i18n`.
- **Тема** — `js/theme.js` переключает класс `dark-theme` на `<html>`,
  а в `style.css` для этого класса заданы свои значения CSS-переменных.

## Если что-то не работает

Скопируй текст ошибки из терминала (или из консоли браузера — F12 → Console)
и разберёмся по шагам, что пошло не так.
